# User Onboarding Checklist

## Purpose
This checklist ensures that all new employees are properly onboarded with the necessary system access, tools, and security configurations required to perform their job responsibilities.

## Scope
Applies to all new hires within the organization requiring access to company systems and resources.

---

## 1. Pre-Onboarding Preparation
- [ ] Receive new hire request from HR or hiring manager
- [ ] Verify employee details (full name, job title, department, start date)
- [ ] Confirm required applications and access level
- [ ] Assign appropriate role/group based on job function

---

## 2. Account Creation (Microsoft 365 / Entra ID)
- [ ] Create user account in Entra ID
- [ ] Assign company email address
- [ ] Set temporary password
- [ ] Enable account

---

## 3. License & Access Assignment
- [ ] Assign Microsoft 365 license
- [ ] Add user to appropriate security groups
- [ ] Grant access to required applications (Teams, SharePoint, OneDrive, etc.)
- [ ] Assign role-based permissions (RBAC)

---

## 4. Security Configuration
- [ ] Enforce Multi-Factor Authentication (MFA)
- [ ] Configure password reset options (SSPR if applicable)
- [ ] Apply conditional access policies (if applicable)
- [ ] Review security baseline compliance

---

## 5. Device & Workstation Setup
- [ ] Prepare company device (if applicable)
- [ ] Install required software and applications
- [ ] Join device to domain / Azure AD
- [ ] Configure email and Teams access

---

## 6. User Communication
- [ ] Send welcome email with login credentials
- [ ] Provide initial login instructions
- [ ] Share IT support contact information
- [ ] Provide basic security awareness guidance

---

## 7. First Login Support
- [ ] Confirm successful login
- [ ] Assist with MFA setup
- [ ] Verify access to required systems
- [ ] Address any initial issues

---

## 8. Documentation & Ticket Closure
- [ ] Document onboarding actions in ticketing system
- [ ] Attach relevant notes and confirmations
- [ ] Mark onboarding ticket as completed
- [ ] Notify manager of completion

---

## Notes
- All onboarding activities must follow company security policies and least privilege principles.
- Any access outside standard roles must be approved by management.
