# User Offboarding Checklist

## Purpose
This checklist ensures that all departing employees have their access to company systems securely removed to prevent unauthorized access and protect organizational data.

## Scope
Applies to all employees, contractors, or vendors leaving the organization.

---

## 1. Offboarding Request & Verification
- [ ] Receive termination or departure request from HR or manager
- [ ] Confirm employee details (name, role, department, last working day)
- [ ] Identify all systems and applications the user has access to

---

## 2. Account Deactivation (Microsoft 365 / Entra ID)
- [ ] Disable user account in Entra ID
- [ ] Block sign-in immediately
- [ ] Reset password
- [ ] Revoke active sessions

---

## 3. Access Removal
- [ ] Remove user from all security groups
- [ ] Revoke access to applications (Teams, SharePoint, OneDrive, etc.)
- [ ] Remove administrative privileges (if applicable)
- [ ] Disable VPN or remote access

---

## 4. License & Resource Management
- [ ] Remove Microsoft 365 license
- [ ] Transfer ownership of files (OneDrive, SharePoint)
- [ ] Archive mailbox if required
- [ ] Set email forwarding (if approved)

---

## 5. Device & Asset Recovery
- [ ] Collect company-issued devices (laptop, phone, badge)
- [ ] Remove device from management system (Intune/Azure AD)
- [ ] Wipe or reset device if necessary

---

## 6. Security & Compliance Actions
- [ ] Review recent account activity for anomalies
- [ ] Document any security concerns
- [ ] Ensure compliance with data retention policies
- [ ] Confirm all access has been revoked

---

## 7. Documentation & Ticket Closure
- [ ] Update ticket with all completed actions
- [ ] Attach relevant logs or notes
- [ ] Mark offboarding ticket as completed
- [ ] Notify HR and manager

---

## Notes
- Offboarding should be completed on or before the employee’s last working day.
- Immediate action is required for involuntary terminations.
- All actions must follow least privilege and security best practices.
