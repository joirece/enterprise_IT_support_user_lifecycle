# Password Reset & Account Recovery SOP

## Purpose
This procedure outlines the secure process for resetting user passwords and assisting with account recovery while ensuring identity verification and compliance with security policies.

## Scope
Applies to all employees requesting password resets or experiencing account access issues.

---

## 1. Request Intake
- [ ] Receive password reset request via ticket, email, or phone
- [ ] Create or update help desk ticket
- [ ] Record user details (name, department, username/email)

---

## 2. Identity Verification
- [ ] Verify user identity using approved methods:
  - Employee ID or username
  - Registered phone number or email
  - Manager confirmation (if required)
- [ ] Do NOT proceed if identity cannot be verified

---

## 3. Account Status Check
- [ ] Check if account is locked, disabled, or active
- [ ] Identify reason for lockout (failed attempts, policy enforcement, etc.)
- [ ] Confirm no suspicious activity is present

---

## 4. Password Reset Process
- [ ] Reset password in Microsoft 365 / Entra ID
- [ ] Assign temporary password (if required)
- [ ] Force password change at next login
- [ ] Unlock account (if locked)

---

## 5. MFA & Security Verification
- [ ] Confirm Multi-Factor Authentication (MFA) is enabled
- [ ] Assist user with MFA setup or reconfiguration if needed
- [ ] Verify recovery methods (SSPR options)

---

## 6. User Communication
- [ ] Provide temporary password securely
- [ ] Instruct user to change password upon login
- [ ] Remind user not to share credentials
- [ ] Provide basic password security guidelines

---

## 7. Post-Reset Validation
- [ ] Confirm user can successfully log in
- [ ] Verify access to required applications
- [ ] Ensure no additional issues are present

---

## 8. Documentation & Ticket Closure
- [ ] Document actions taken in ticket
- [ ] Note identity verification method used
- [ ] Mark ticket as resolved
- [ ] Escalate if any suspicious activity is detected

---

## Security Notes
- Never share passwords in plain text over unsecured channels
- Always verify user identity before resetting passwords
- Report any suspected account compromise immediately
- Follow least privilege and security best practices at all times
